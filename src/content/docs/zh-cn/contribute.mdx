我们欢迎任何大小的贡献和任何技能水平的贡献者。作为一个开源项目，我们相信应该回馈给我们的贡献者。我们很乐意提供指导，帮助你完成 PR，进行技术写作，以及将任何想法变成现实。

## 贡献的方式

### 项目仓库

这里有很多方式可以为 **统一场论文档** 项目做出贡献！每个 **统一场论文档** 仓库都包含了一个 README 文件，在其末尾会链接到一个 `CONTRIBUTING.md` 文件。

你可以通过访问 [统一场论 的 GitHub 主页](https://github.com/unified-field-theory-org) 来找到以下仓库：

-  [统一场论的主要代码库](https://github.com/unified-field-theory-org/unifiedfieldtheory.cn)，包括官方集成和脚手架的模板。
-  [统一场论资源文档](https://github.com/unified-field-theory-org/documentation)，统一场论的资源文档仓库！不仅可以贡献书面内容，还可以贡献涉及无障碍性、CSS、UI 和 UX 的 Astro 代码。我们还将文档翻译成多种语言，因此我们需要你的帮助将整个网站翻译成其他语言。
-  [统一场论演示动画](https://github.com/unified-field-theory-org/animations)，统一场论的演示动画，使用 [manim](https://github.com/3b1b/manim) 制作。

### 贡献的类型

除了贡献自己的代码或内容外，你还可以通过参与评审 PR 的过程、在现有的 GitHub Issues 和讨论中提出想法，以及参与我们的「置顶」问题维护任务来做出巨大的贡献，你的参与对整个统一场论社区来说非常重要！

我们还有非常活跃的 [论坛](https://forum.unifiedfieldtheory.cn) / [Slack](https://unifiedfieldtheorycn.slack.com) / [Discord](https://discord.gg/W3JYMwBBWb) 社区！我们非常重视那些帮助维护我们社区健康的社区成员。

## 我们的贡献者

这些文档由所有这些有帮助的人提供。[在 GitHub 上加入我们！](https://github.com/unified-field-theory-org/unifiedfieldtheory.cn)
